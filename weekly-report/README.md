# Weekly Productivity Report

Generates a self-contained HTML report of your weekly output:
- Jira tickets resolved and in-progress
- Azure DevOps PRs you created (merged / open)
- Real code commits (merge commits excluded)
- AI-native metrics — which PRs were AI-assisted and by which tool

---

## Quick Start

```bash
cd ~/Documents/AI-Agent

# Generate report for last week (auto-selected on Mondays)
node weekly-report/weekly-report.mjs

# Generate report for current week
node weekly-report/weekly-report.mjs --week 2026-W16

# View all saved reports
node weekly-report/weekly-report.mjs history

# Re-open a past report in the browser
node weekly-report/weekly-report.mjs open 2026-W15
```

The report opens automatically in your browser. No extra setup — Jira and Azure DevOps credentials are read from existing config files.

---

## All Commands

| Command | What it does |
|---|---|
| `node weekly-report.mjs` | Current week report (last week on Mondays) |
| `node weekly-report.mjs --last` | Last completed week, regardless of day |
| `node weekly-report.mjs --week 2026-W15` | Any specific ISO week |
| `node weekly-report.mjs history` | List all saved snapshots |
| `node weekly-report.mjs open 2026-W15` | Open a past HTML report in browser |
| `node weekly-report.mjs --no-open` | Generate report without auto-opening browser |
| `node weekly-report.mjs --help` | Show help |

Week numbers follow ISO 8601 — weeks start on Monday.

## Credentials

**Jira** — reads automatically from `jira-creator/jira-config.json`. No setup needed.

**Azure DevOps** — resolved in this priority order:
1. `AZURE_TOKEN` environment variable
2. `azureDevOps.pat` in `Dr.-Nexus/config.json`
3. `_ado_token` shell function (sourced from `~/.zshrc`) ← **auto-used if configured**

Since your config has `tokenCommand: "_ado_token"`, Azure DevOps works automatically without setting any env var.

## What the Report Shows

### Summary Cards
Four cards at the top — with week-over-week delta arrows once two weeks are saved:
- **Tickets Resolved** — Jira issues you closed this week (with story points)
- **PRs Merged** — Azure DevOps PRs you created that were merged
- **Commits** — Real code commits (merge commits stripped out)
- **AI Assist Rate** — % of your PRs that contain an AI tool signature

### Jira Section
- **Resolved this week** — issues where `resolutiondate` falls in the week window
- **In Progress** — issues assigned to you in `indeterminate` or `new` status

### Pull Requests
Every PR you created this week (active + merged; abandoned ones excluded):
- Repo name, PR number (linked to Azure DevOps), status badge
- AI badge if the PR description contains a known AI signature

### AI-Native Metrics
- Total AI-assisted PRs, assist rate, Cursor commits
- Tool breakdown bar chart (Claude / Codex / Cursor)
- Commit activity sparkline by day

## How AI Detection Works

The report scans the PR description for signatures that AI tools add automatically:

| What it looks for in PR description | Detected as |
|---|---|
| `Co-Authored-By: Claude ...` | **Claude** |
| `🤖 Generated with [Claude Code]` | **Claude** |
| `Auto-generated by NEXUS` | **Claude** (Dr.-Nexus is Claude-powered) |
| `made-with: cursor` | **Cursor** |
| `Co-Authored-By: ... cursor` | **Cursor** |
| `codex` / `openai` / `gpt-4` | **Codex** |

No signature in the PR description = not counted as AI-assisted.

**Dr.-Nexus PRs** are automatically detected as Claude from `2026-W16` onwards.  
For older PRs, add `_ado_token` to the zshrc and re-run any week to backfill.

## Annual Portfolio

The weekly snapshots feed the annual portfolio. To regenerate the full portfolio:

```bash
# 1. Pull full-year AI metrics from Azure DevOps
AZURE_TOKEN=$(_ado_token) \
  FROM_DATE=2025-01-01T00:00:00Z \
  node utils/ai-metrics.js

# 2. Fetch all Jira tickets (365 days)
node scripts/jira-review-dashboard.mjs

# 3. Build the portfolio HTML
node scripts/jira-evidence-portfolio.mjs
# → opens review-data/work-evidence-portfolio.html
```
